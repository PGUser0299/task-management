## 1. データモデル詳細

### 1.1 ER 図（詳細）

```

    title             VARCHAR(255)  NOT NULL
    description       TEXT  BLANK
    status            VARCHAR(20)  CHOICES[pending|todo|in_progress|done]  DEFAULT 'pending'
    priority          VARCHAR(20)  CHOICES[low|medium|high|urgent]  DEFAULT 'medium'
```

### 1.2 フィールド定義詳細

#### Task.status

| 値 | ラベル | 説明 |
|---|---|---|
| `pending` | Pending | 未着手（デフォルト） |
| `todo` | To Do | 着手予定 |
| `in_progress` | In Progress | 作業中 |
| `done` | Done | 完了 |

#### Task.priority

| 値 | ラベル | AI ソート順（昇順） |
|---|---|---|
| `urgent` | Urgent | 0（最優先） |
| `high` | High | 1 |
| `medium` | Medium | 2 |
| `low` | Low | 3 |

#### Task.estimate_minutes

- 単位: **分**（API の入出力とも分）
- フロントエンド表示: `Math.ceil(minutes / (60 * 8))` で **日** 換算表示
- フロントエンド入力: 日数 → `days * 8 * 60` 分に変換して送信
- 1日 = 8時間 = 480分 と定義

#### TeamMembership.role

| 値 | 権限 |
|---|---|
| `owner` | チームの全操作（メンバー管理を含む） |
| `member` | チームリソースの閲覧・タスク操作 |

## 2. API 詳細仕様

### 2.1 共通仕様

- **Base URL**: `http://localhost:8000/api`（開発）
- **Content-Type**: `application/json`
- **認証ヘッダー**: `Authorization: Bearer {access_token}`
- **ページネーション**: `?page=N` クエリパラメータ、レスポンスは `{count, next, previous, results}` 形式

---

### 2.2 認証 API

#### POST `/api/auth/register/`

新規ユーザー登録。

**リクエスト**

```json
{
  "username": "alice",
  "password": "securepass123",
  "email": "alice@example.com",     // 任意
  "display_name": "Alice"           // 任意
}
```

**レスポンス** `201 Created`

```json
{
  "id": 1,
  "username": "alice",
  "email": "alice@example.com",
  "display_name": "Alice"
}
```

**バリデーション**

| フィールド | ルール |
|---|---|
| username | 必須、150文字以内、英数字・@/./+/-/_ のみ、一意 |
| password | 必須、8文字以上、一般的なパスワード不可、数字のみ不可 |

---

#### POST `/api/auth/login/`

ログイン（JWT 取得）。

**リクエスト**

```json
{ "username": "alice", "password": "securepass123" }
```

**レスポンス** `200 OK`

```json
{
  "access":  "eyJ0eXAiOiJKV1QiLCJhb...",
  "refresh": "eyJ0eXAiOiJKV1QiLCJhb..."
}
```

---

#### POST `/api/auth/token/refresh/`

アクセストークンをリフレッシュ。

**リクエスト**

```json
{ "refresh": "eyJ0eXAiOiJKV1QiLCJhb..." }
```

**レスポンス** `200 OK`

```json
{ "access": "eyJ0eXAiOiJKV1QiLCJhb..." }
```

---

#### GET `/api/auth/me/`

ログイン中のユーザー情報取得。`is_admin` は Django Group `管理者` 所属（または `is_superuser=True`）であれば `true`。フロントエンドはこの値でチーム作成・プロジェクト作成・メンバー追加ボタンの表示を切り替える。

**レスポンス** `200 OK`

```json
{
  "id": 1,
  "username": "alice",
  "email": "alice@example.com",
  "display_name": "Alice",
  "is_admin": false
}
```

---

#### PATCH `/api/auth/me/display-name/`

ログイン中のユーザーの表示名を変更する。本人確認のため現在のパスワードが必要。

**リクエスト**

```json
{
  "display_name": "新しい表示名",
  "password": "currentPassword123"
}
```

**レスポンス** `200 OK`

```json
{
  "id": 1,
  "username": "alice",
  "email": "alice@example.com",
  "display_name": "新しい表示名"
}
```

**バリデーション**

| フィールド | ルール |
|---|---|
| display_name | 必須、150文字以内 |
| password | 必須、現在のパスワードと一致すること |

---

#### POST `/api/auth/me/password/`

ログイン中のユーザーのパスワードを変更する。

**リクエスト**

```json
{
  "old_password": "oldPassword123",
  "new_password": "newPassword456",
  "new_password_confirm": "newPassword456"
}
```

**レスポンス** `200 OK`

```json
{ "detail": "パスワードを変更しました。" }
```

**バリデーション**

| フィールド | ルール |
|---|---|
| old_password | 必須、現在のパスワードと一致すること |
| new_password | 必須、8文字以上 |
| new_password_confirm | 必須、new_password と一致すること |

### 2.6 AI API

#### POST `/api/ai/parse-task/`

自然言語テキストからタスク情報を抽出する。

**リクエスト**

```json
{
  "text": "来週のリリース準備（テスト、ドキュメント、リリースノート作成など）"
}
```

**レスポンス** `200 OK`

```json
{
  "title": "来週のリリース準備（テスト、ドキュメント、リリースノート作成など）",
  "description": "来週のリリース準備（テスト、ドキュメント、リリースノート作成など）",
  "estimate_minutes": 90,
  "priority": "medium",
  "subtasks": [
    { "title": "テスト",                 "estimate_minutes": 30 },
    { "title": "ドキュメント",           "estimate_minutes": 30 },
    { "title": "リリースノート作成など", "estimate_minutes": 30 }
  ]
}
```

**ヒューリスティクス処理ロジック**

| 条件 | 優先度 |
|---|---|
| テキストに "urgent" / "緊急" / "至急" / "critical" を含む | `urgent` |
| テキストに "today" / "本日" / "今日" を含む | `high` |
| テキストに "someday" / "いつか" / "後で" を含む | `low` |
| その他 | `medium` |

見積: サブタスク数 × 30分（サブタスクなしの場合は 60分固定）

---

#### POST `/api/ai/suggest-today/`

今日フォーカスすべきタスクを提案する。

**リクエスト**

```json
{
  "team_id": 1,         // 任意
  "project_id": 1,      // 任意
  "limit": 5            // 1〜20、デフォルト5
}
```

**レスポンス** `200 OK`

```json
{
  "summary": "今日フォーカスすべきタスクをピックアップしました。うち 2 件は優先度 High 以上です。",
  "items": [
    {
      "id": 5,
      "project_id": 1,
      "title": "API 設計",
      "status": "in_progress",
      "priority": "high",
      "due_date": "2026-04-12",
      "reason": "本日が期限のタスクです。優先度が high に設定されています。"
    }
  ]
}
```

**ソートロジック**

1. `status` が `pending` / `todo` / `in_progress` かつ `parent__isnull=True`（サブタスク除外）
2. `due_date` 昇順（期限が近い順、NULL は後ろ）
3. 優先度昇順（`urgent:0` > `high:1` > `medium:2` > `low:3`）

**reason 生成ロジック**

| 条件 | テキスト |
|---|---|
| 期限超過（due_date < today） | 「期限超過のタスクです。」 |
| due_date == today | 「本日が期限のタスクです。」 |
| due_date <= today + 2日 | 「期限が数日以内に迫っています。」 |
| priority が high または urgent | 「優先度が {priority} に設定されています。」 |
| 上記いずれも該当しない | 「進行中または未着手のタスクの中からバランスよく選ばれました。」 |

## 3. 認証・認可詳細

### 3.1 JWT フロー

```
[フロントエンド]                          [バックエンド]
      │                                        │
      │─ POST /api/auth/login/ ───────────────▶│
      │                                        │ SimpleJWT でユーザー認証
      │◀─ { access, refresh } ────────────────│
      │                                        │
      │  localStorage.setItem('taskapp_tokens',│
      │    { access, refresh })                │
      │                                        │
      │─ GET /api/tasks/ ─────────────────────▶│
      │  Authorization: Bearer {access}        │ JWTAuthentication で検証
      │                                        │
      │◀─ 200 OK  (tasks data) ───────────────│
      │                                        │
      │─ GET /api/tasks/ (access 期限切れ) ───▶│
      │  Authorization: Bearer {expired}       │
      │◀─ 401 Unauthorized ───────────────────│
      │  window.dispatchEvent(                 │
      │    'taskapp:unauthorized')             │
      │  localStorage.removeItem(...)          │
      │  Navigate → /login                     │
```

### 3.2 権限クラス詳細

#### `IsTeamMember` （`tasks/permissions.py`）

```
has_permission(request, view)
  └─ return request.user.is_authenticated
       （list/create は get_queryset でチームフィルタ済）

has_object_permission(request, view, obj)
  ├─ obj が Project  → obj.team
  ├─ obj が Task     → obj.project.team
  ├─ obj が TaskComment → obj.task.project.team
  └─ obj が Team     → obj
       └─ team.memberships.filter(user=request.user).exists()
```

#### `IsAdminGroupMemberForWrite` （`users/permissions.py`）

```
has_permission(request, view)
  ├─ 未認証           → False
  ├─ SAFE_METHODS    → True（GET/HEAD/OPTIONS は全員許可）
  └─ それ以外        → is_admin_group_member(request.user)
       ├─ user.is_superuser                         → True
       └─ user.groups.filter(name='管理者').exists() → True/False
```

- 適用先: `TeamViewSet`、`ProjectViewSet`
- `message`: 「この操作は管理者グループのメンバーのみ実行できます。」
- `管理者` Django Group は `users/migrations/0002_admin_group.py` のデータマイグレーションで自動生成される。

#### チームオーナーチェック（`users/views.py` TeamMembershipViewSet）

```
_require_owner(team):
  team.memberships.filter(user=request.user, role='owner').exists()
  → False の場合 PermissionDenied(403) を raise
```

#### `UserViewSet` の候補絞り込み（`users/views.py`）

```
get_queryset():
  └─ User.objects.exclude(is_superuser=True).order_by('id')
       （チームメンバー・担当者候補から Django の superuser を除外）
```

### 3.3 シリアライザバリデーション

#### `ProjectSerializer.validate_team_id`

```
team.memberships.filter(user=request.user).exists()
→ False の場合 ValidationError(400)
```

#### `TaskSerializer.validate`

```
if parent is not None and project is not None:
    parent.project_id != project.pk
    → True の場合 ValidationError(400)
```

## 4. バックエンド内部設計

### 4.1 アプリ構成と責務

```
config/          ─ Django プロジェクト設定・グローバル URL
users/           ─ ユーザー認証・チーム管理
  models.py      ─ User(AbstractUser), Team, TeamMembership
  serializers.py ─ 登録・チーム・メンバーシリアライザ
                   UpdateDisplayNameSerializer（display_name + password検証）
                   ChangePasswordSerializer（old_password / new_password / confirm）
                   MeSerializer（/auth/me/ 用、is_admin フラグを付与）
  permissions.py ─ IsAdminGroupMemberForWrite 権限クラス、
                   is_admin_group_member() ヘルパー（管理者グループ判定）
  views.py       ─ 認証 View + チーム/メンバー ViewSet
                   UpdateDisplayNameView (PATCH /auth/me/display-name/)
                   ChangePasswordView    (POST  /auth/me/password/)
                   TeamViewSet には IsAdminGroupMemberForWrite を適用
                   UserViewSet は is_superuser=True を除外
  migrations/0002_admin_group.py
                 ─ Django Group『管理者』をデータマイグレーションで作成
  urls_auth.py   ─ /api/auth/ 配下
                   register/, login/, token/refresh/, me/,
                   me/display-name/, me/password/
  urls.py        ─ /api/ 配下（teams/team-memberships/users）

tasks/           ─ プロジェクト・タスク管理
  models.py      ─ Project, Task, TaskComment
  serializers.py ─ プロジェクト・タスク・コメントシリアライザ
  views.py       ─ Project/Task/TaskComment ViewSet
                   ProjectViewSet には IsAdminGroupMemberForWrite を併用
  permissions.py ─ IsTeamMember 権限クラス
  urls.py        ─ /api/ 配下（projects/tasks/comments）

ai_assist/       ─ AI タスク支援
  services.py    ─ TaskAIService（ヒューリスティクスロジック）
  serializers.py ─ リクエスト/レスポンスシリアライザ
  views.py       ─ ParseTaskView, SuggestTodayView
  urls.py        ─ /api/ai/ 配下
```

### 4.2 URL 構成

```
/api/auth/
  ├── register/
  ├── login/
```

#### MemberAddDialog 候補絞り込みロジック

```
candidates = users  // GET /api/users/ の結果（サーバー側で superuser 除外済）
  .filter(u => !u.is_superuser)                   // 念のためクライアント側でも除外
  .filter(u => !existingMemberIds.has(u.id))      // 既に所属しているユーザーも除外
```

## 7. エラーハンドリング

### 7.1 バックエンドのエラーレスポンス形式

| HTTP ステータス | 場面 | レスポンス例 |
|---|---|---|
| 400 Bad Request | バリデーションエラー | `{"username": ["このフィールドは必須です。"]}` |
| 401 Unauthorized | 認証失敗・トークン期限切れ | `{"detail": "No active account found with the given credentials"}` |
| 403 Forbidden | 権限不足 | `{"detail": "この操作はチームのオーナーのみ実行できます。"}` |
| 404 Not Found | リソース不存在 | `{"detail": "Not found."}` |

### 7.2 フロントエンドのエラーハンドリング

```
Axios Response Interceptor
  └─ status 401
      ├─ localStorage.removeItem('taskapp_tokens')
      └─ window.dispatchEvent('taskapp:unauthorized')
          └─ AuthContext で受信 → setTokens(null)
              └─ PrivateRoute で /login にリダイレクト

各コンポーネント
  ├─ isLoading  → CircularProgress 表示
  ├─ isError    → エラーメッセージ表示
  └─ mutate エラー → extractApiError(err) でインライン表示
```

### 7.3 extractApiError ユーティリティ

**ファイル**: `src/lib/apiError.ts`

DRF が返す複数形式のエラーレスポンスを統一された文字列に変換する共有ユーティリティ。

```typescript
// 対応する DRF エラー形式
{ detail: "..." }                         // → detail の値
{ non_field_errors: ["..."] }             // → 先頭要素
{ field_name: ["..."] }                   // → 最初のフィールドの先頭要素
"エラーメッセージ文字列"                  // → そのまま
// 上記いずれでもない → '予期しないエラーが発生しました。'
```

使用箇所: `DisplayNameForm`, `PasswordForm`, `LoginPage`, `RegisterPage`

### 7.4 ErrorBoundary

**ファイル**: `src/components/ErrorBoundary.tsx`

React クラスコンポーネント。認証済みルート全体（`<Outlet />` ラッパー）に適用され、コンポーネントのレンダリングエラーをキャッチする。

```
ComponentDidCatch(error, info)
  └─ this.setState({ hasError: true })
      └─ フォールバック UI 表示
          ├─ エラーメッセージ
          └─ 「ページを再読み込み」ボタン → window.location.reload()
```

### 7.5 RegisterPage エラー優先順位

```
RegisterErrorData = {
  username?:          string[]
  email?:             string[]
  password?:          string[]
  non_field_errors?:  string[]

表示優先: username[0] > email[0] > password[0] > non_field_errors[0] > '登録に失敗しました。'
```
