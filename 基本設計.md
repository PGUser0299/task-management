## 1. システム概要

### 1.1 目的

チームで共有するタスク管理ツール。複数チーム・複数プロジェクトにまたがるタスクを一元管理し、AI によるタスク解析・今日の優先タスク提案によって作業効率を向上させる。

### 1.2 システム名

**AI チームタスクボード** (TeamTaskBoard)

### 1.3 対象ユーザー

- チームのオーナー（チーム・メンバー管理）
- チームのメンバー（タスク操作）

### 1.4 利用シナリオ概要

```
ユーザー登録 → チーム作成 → プロジェクト作成
    → タスク作成（手動 or AI自然言語分解）
    → カンバンボードでステータス管理
    → ダッシュボードで AI 今日サジェスト確認
    → アカウント設定で表示名・パスワード変更
```

## 7. 認証・認可方針

### 7.1 認証方式

- **JWT (JSON Web Token)** を採用
- アクセストークン有効期限: **30 分**
- リフレッシュトークン有効期限: **7 日**
- トークンは フロントエンドの `localStorage` に保存（`taskapp_tokens` キー）

### 7.2 認証フロー概要

```
1. POST /api/auth/login/ → access token + refresh token 取得
2. 以後のリクエストに Authorization: Bearer {access} ヘッダーを付与
3. access token が 401 → localStorage をクリア → ログイン画面へ
4. フロントエンドはリフレッシュ機能を持つ（POST /api/auth/token/refresh/）
```

### 7.3 認可方針

- **オブジェクトレベル認可**: `IsTeamMember` 権限クラスがチーム所属を確認
- **チームオーナー操作**: `TeamMembershipViewSet` でオーナーのみ許可
- **コメントの所有者チェック**: 自分が作成したコメントのみ編集・削除可
- **プロジェクト作成**: シリアライザで指定チームへの所属を検証
- **サブタスク**: 同一プロジェクト内のタスクのみ親に指定可
- **表示名変更**: パスワード確認によるセルフ認証（`UpdateDisplayNameSerializer` で検証）
- **パスワード変更**: 現在のパスワード確認後に新パスワードを設定

## 8. 非機能要件

### 8.1 パフォーマンス

| 項目 | 方針 |
|---|---|
| N+1 クエリ対策 | `select_related` を ViewSet の `get_queryset` で適用 |
| ページネーション | デフォルト 20 件/ページ（`PageNumberPagination`） |
| フロントエンドキャッシュ | React Query のクエリキャッシュで重複リクエストを抑制 |
| 再レンダリング最適化 | 派生状態に `useMemo` を適用（DashboardPage・ProjectTasksPage） |

### 8.2 セキュリティ

| 項目 | 方針 |
|---|---|
| シークレット管理 | `django-environ` で環境変数から読み込み（`.env` は git 管理外） |
| CORS | 本番は `CORS_ALLOWED_ORIGINS` で許可オリジンを限定 |
| パスワード | Django 標準バリデーター4種を適用、変更時は現在パスワードを必須確認 |
| HTTPS | 本番環境ではリバースプロキシ（Nginx 等）で終端 |

### 8.3 信頼性・保守性

| 項目 | 内容 |
|---|---|
| エラーバウンダリ | `ErrorBoundary` コンポーネントを認証済みレイアウト全体に適用 |
| エラーメッセージ共通化 | `src/lib/apiError.ts` の `extractApiError()` で一元管理 |
| コンポーネント設計 | 単一責任原則に従いフォームコンポーネントを分割（Profile / Subtask） |
| 共有定数 | `taskConstants.ts` に STATUS_OPTIONS を集約し重複を排除 |

### 8.4 開発・運用

| 項目 | 内容 |
|---|---|
| 開発環境 DB | SQLite（`.env` の `DATABASE_URL` で切替） |
| 本番環境 DB | PostgreSQL（`psycopg2-binary` 導入済） |
| フロント開発サーバー | `npm run dev`（Vite HMR） |
| バックエンド開発サーバー | `python manage.py runserver` |
| 型チェック | TypeScript strict mode（`tsc --noEmit` でゼロエラー確認済） |
| Django システムチェック | `manage.py check` でゼロエラー確認済 |
